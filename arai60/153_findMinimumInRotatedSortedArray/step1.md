### 思考記録
二分探索の問題と知った状態で問題文を読んでしまっているので、これもなにか評価式に当てはめて二分していくんだろうなと考える。
先頭を探すのに愚直にmidを取ってなんらかの評価式に当てはめても、それだけでずらされた先頭を探すのは難しそうという気持ちになる。
残り使える情報はなんだろうと考えたときに、先頭とか末尾とかを見たらオーダーがどうなっているかわかりそうと感じる。
先頭とmidと末尾の大小関係を見ていけば、ずらされた先頭がわかりそうな気持ちになったので、一旦コードを書いてみる。

### 計算量
#### 時間計算量
O(logN)
#### 空間計算量
O(1)

### コード
```ruby
# @param {Integer[]} nums
# @return {Integer}
def find_min(nums)
    left = 0
    right = nums.length - 1
    while left < right
        mid = (left + right) / 2

        if nums[mid] < nums[right]
            right = mid
        else
            left = mid + 1
        end
    end

    nums[left]
end
```
