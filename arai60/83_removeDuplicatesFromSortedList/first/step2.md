### 思考記録
同じ値である限りはその場でListNodeを置き換えて、値が増えたらListNodeのnextを次の値にして、次の値でまた同じ作業を繰り返すイメージでコードを書く。
同じ値だと足踏みしてその場ループ、次の値が来ると一歩進むみたいなイメージ。

nextが予約語なのに変数名で使ってしまって構文エラー。

"同じ値だと足踏みしてその場ループ、次の値が来ると一歩進むみたいなイメージ。"このイメージだとだめだな。
最初にheadは固定してnextの参照を変えていく必要があるから、nextは同じ値でも違う値でもどちらでも参照を変えてよくて、nextを更新する対象を移動するのは値が変わったときみたいなイメージか。
nextがないときはLinkedListの終わりであることは不変条件。

いやLinkedListはnodeのつながりを表していることを考えるとその場ループとかはわけが分からない発想だ。
隣接nodeが同じ値ならlinkから外して、さらに次のnodeに繋ぎ変えるという捉え方が自然な気がする。
そして、繋ぎ変えたあとにまた隣接nodeを見て同じなら繋ぎ変える。
つまり、ここまでは重複を削除しているというcursorを置いて、cursorの隣接ノードを確認し、同じ値であれば隣接ノードをさらに隣に付け替えて、違う値になったらそこが新しい重複削除の完了地点になるからcursorを更新する。
cursorの隣接ノードがnilになるまで続ければよい。
これな気がする。

### 計算量
#### 時間計算量
O(N)
#### 空間計算量
O(1)

### コード
#### 提出コード
```ruby
def delete_duplicates(head)
    return head if head.nil?

    dedup_tail = head
    while (tail_candidate = dedup_tail.next)
        if dedup_tail.val == tail_candidate.val
            dedup_tail.next = tail_candidate.next
        else
            dedup_tail = tail_candidate
        end
    end

    head
end
```
