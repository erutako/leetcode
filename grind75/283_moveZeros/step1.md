### やろうとしていることを言語化してみる
数字が書かれたカードが並んでいて、ゼロが書かれたカードを右端に寄せたい。

### 思考記録
数字の配列が与えられる。
0以外の数字の並び順を維持しながら、すべてのゼロを末尾に動かす。
配列のコピーは作っちゃだめで、in-placeでやる必要がある。

O(N)でやれそうな感覚はある。
0が見つかったらどんどん次の要素と交換していきたい。
0が2以上になったときどうするかだな。

0の数を数えておくと良さそう。
ほんとか？
0が見つかったら次の要素と交換したい。
次の要素が0だったら、その次の要素と0を交換したい。
これを手元でやってみる。

無駄はありそうだけどいけそう。

コード書いてたらO(N^2)になることに気づいた。
一番左の0のindexを持っておいて、そいつと非0の数字を交換するのであればO(N)でいけそう。

indexをもたなくても0の数を保持していたらいけそうとコードを書いてて思った。
0が複数あるときとそうでないときで処理を分けるとうまくいきそう。

なんかよくわからなくなってきた。
とりあえず0にあたったときどうしたいかと0にあたった後0以外にあたったらどうしたいかな気がしてきた。

0にあたったら次の値とスワップしたい。
次の値が0ならなにもしなくて良い。
次の値が非0なら一番先頭の0と交換したい。

rubyの多重代入が内部的にどのようにメモリを使っているかは確認したい気持ち。

答えを見た。
もう一度5分くらいで解く。
えーっと、スワップしたい左端の0の位置を覚えておいて、非0の数字が現れたらその場所と交換してあげる。
交換したら交換先が次のスワップしたい位置でいいんだっけか？
なんかダメそうな気がするけどとりあえず書いてみる。

よくわかんなくなったので8minぐらいでもう一度答え見る。
あーなるほど、書き込み位置は考えていた通りで左端の0の位置を持っている。
0のときは何もせずに次の値を見る。これを続けるとどこかで非0に当たる。
非0にあたったときは左端の0と交換して、次の書き込み位置をとなりのインデックスにずらす。
0が連続している場合はこの操作でスワップ後の左端の0のインデックスを指すことになる。
0が連続していなかった場合は、スワップ後の0のインデックスを指しているので、これまた左端の0のインデックスを指していることになる。
うーん、なんかこの辺がしっくり来てないからすっとコードが書けないんだな。
で、最初非0が連続していたとして、これは書き込み位置がどんどん右にズレていきつつ、書き込み位置にある値をそのまま書き込んでいるだけなので順番に何も影響しない。

もう一度コード書いてみる。
each_with_index, each.with_index, with_indexの使い分けができておらずエラー。
今回はindexのみ知れればよいので、with_indexを使う。


### 計算量
#### 時間計算量
O(N)
#### 空間計算量
O(1)

### コード
```ruby
# @param {Integer[]} nums
# @return {Void} Do not return anything, modify nums in-place instead.
def move_zeroes(nums)
    write = 0
    nums.each_index do |read| 
        next if nums[read].zero?
        nums[write], nums[read] = nums[read], nums[write]
        write += 1
    end
end

```
