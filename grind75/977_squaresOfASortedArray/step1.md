### 思考記録
最初はナイーブに二乗してソートする方法を考える。
普通にソートする時点でO(NlogN)が確定するので、なにかをうまいことしないとダメそう。

入力が配列済みソートであることに気づく。
この性質をうまく使いたい。

0まず見つけて、そこから左がマイナス、右がプラスだから、左と右を同時に走査して値を比べながら新しい配列を作っていけばO(N)でもいけそう。

最初から両端から走査しながら、平方数が大きくなる値を配列のお尻から詰めていけば最初に0を見つける必要もなさそう。
ここまで15min
とりあえず大きな方針はできた。左と右で同じ平方数になるときや最後どうやって終わる？など考えるところはまだありそうだけど、まずはコード書いて具体的な値で動きを見てみる。

コード書いてみると色々rubyでどう書くか調べていたら時間がかかってしまった。
あとは最後の右からのポインタと左からのポインタが同じところを見たときにどうするかがうまくケアできていなくて、一度間違ったoutput出してしまった。
これもプログラムの条件をちゃんと理解できていなかったからな気がする。
最終的にはループの最初でポインタが被っていないか確認して、被っていたらポインタの値を配列の先頭に入れて終わるという理解に落ち着く。

### 計算量
#### 時間計算量
O(N)
#### 空間計算量
O(N)

### コード
```ruby
# @param [Array<Integer>] nums
# @return [Array<Integer>]
def sorted_squares(nums)
    sorted_squares = []
    right = 0
    left = nums.length - 1
    nums.each.with_index(1) do |_num, i|
        return sorted_squares.prepend(nums[right] ** 2) if right == left

        if nums[right].abs >= nums[left].abs
            sorted_squares.insert(-i, nums[right] ** 2)
            right += 1
        else
            sorted_squares.insert(-i, nums[left] ** 2)
            left -= 1
        end
    end
end
```
