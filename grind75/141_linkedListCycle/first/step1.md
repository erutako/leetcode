### 思考記録
問題をみる。
まず設定がよくわからなかった。
posというのは説明用の名前なんだろう。

linked listの各nodeを構成するクラスがあって、それが配列で渡されそう。
valueには値が入っていて、nextが次のnodeのポインタを指しているのだろう。

ぱっと思いつくのは全nodeごとにリストを一周して、自分をnextに指しているnodeを探す。これはN^2。
つぎに2パスでまず全nodeのHashSetを持って、そのあとnodeを1周走査して見つける。これは時間計算量がNだけど空間計算量もNになっちゃうので、微妙。
時間計算量Nは絶対掛かりそう。空間計算量は抑えられそうだけど、全部nodeを候補にして毎回cycleがないか確認するとNになるのでできないか、なにか別の工夫が必要。
とりあえず、時間計算量N、空間計算量Nで実装してみる。

### 計算量
#### 時間計算量
O(N)
#### 空間計算量
O(N)

### コード
```ruby
def hasCycle(head)
    nodes = Set.new
    current = head
    while current
        return true if nodes.include?(current)
        nodes << current
        current = current.next
    end
    return false
end
```
